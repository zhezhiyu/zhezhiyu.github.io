使用多target来构建大量相似App

发表于 2018-08-30

前言

由于公司业务需求, 需要构建大量相似的App, App之间有大量共享的资源, 且有着相似但又不完全相同的功能和界面, 故采用多Target编译方案来实现。

技术解决方案

分析现有项目, 确定解决方案主要分为3步.
  1. 从现有项目中抽取子项目, 构建可复用的大模块.
  2. 通过多Target编译的方式, 不同的Target在编译时, 采用不同的资源文件和源文件.
  3. 在第2步的基础上, 在项目中创建配置用的Config类, 然后在不同Target各自的配置文件中设置不同的Config值, 来实现差异化界面.

抽取子项目

从现有项目中, 将可重用的模块抽取出来, 以git submodule 的形式组织到项目中. 主要抽取的submodule为:
  1. Core Common, 涉及可复用的底层模块. 包括网络请求模块, 缓存模块, 推送模块, 分享模块, 项目中核心算法, 静态util方法等.
  2. UI Common, 涉及可复用的注册登录界面, 充值支付界面, 意见反馈界面, 关于界面, 彩种详情界面等. 另外, 将一些可复用的UI风格控件抽取成相应的静态工厂方法, 用于生成同一风格的按钮、背景及状态栏等.
  3. Lib Common, 所有第三方的开源库依赖
  4. Resources Common, 共享图片等资源
